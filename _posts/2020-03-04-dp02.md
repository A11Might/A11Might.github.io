动态规划（简称DP），它的思路是保存计算过的子问题的答案以重复使用，来避免重复计算。

## 一、题目特点

大部分动态规划题属于以下三类，分别使用数塔问题举例：

有如下所示的数塔，要求从顶层走到底层，若每一步只能走到相邻的结点。

![数塔问题](../img/post/dp/数塔问题.png){: .center-block :}

1. 最值：求走出一条路径的最大数字和是多少。
2. 计数：求走出一条路径数字和不超过 55 的方案数。
3. 存在性判定：求是否存在一条路径的数字和等于 55。

对于 1、3 情况，你应该想想是否有更简单的贪心算法。

## 二、代码实现

有两种实现方法：

- 自上而下 - 记忆化搜索（递归）

记忆化搜索是一种相对简单的动态规划实现方法，我们可以很容易的实现递归的方法，然后通过一个额外的数组保存计算过的子问题的答案，当递归到某个子问题时，如果答案计算过就直接返回，否则就递归计算。

记忆化搜索不用考虑状态之间的计算顺序，程序只需要考虑当前问题和当前问题的子问题即可。

- 自下而上 - 递推（迭代）

递推使用嵌套的 For 循环来枚举所有的状态，并使用状态转移方程更新状态。优点是开销小（没有记忆化搜索中系统栈的开销）、运行时间短。代码通常简短清晰，容易看出时空复杂度。在较难问题中，可以加入各种动态规划优化。

递推需要考虑状态之间的计算顺序（即枚举状态的顺序），比如我们更新状态 A 时，需要状态 B、C、D 的答案，那么状态 B、C、D 就必须先于状态 A 被我们枚举到。

## 三、解题步骤

大致可以分为 4 步：

1. 确定状态表示，包括阶段划分（状态从一个阶段转移到另一个阶段）与状态表示。
2. 写出转移方程：帮助你想清楚状态之间是如何转移的。
3. 确定边界：base case。
4. 如果使用递推，需要考虑状态枚举的顺序。

例题：[64. 最小路径和](https://leetcode-cn.com/problems/minimum-path-sum/)

> 给定一个包含非负整数的 *m* x *n* 网格，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。
>
> **说明：**每次只能向下或者向右移动一步。
>
> **示例:**
>
> ```
> 输入:
> [
> [1,3,1],
> [1,5,1],
> [4,2,1]
> ]
> 输出: 7
> 解释: 因为路径 1→3→1→1→1 的总和最小。
> ```

先确定状态表示，从起点位置可以向下或者向右移动，所以我们如下划分阶段：阶段 1 是距离起点 0 步的位置；阶段 2 是距离起点 1 步的位置……我们总是从上一个阶段的状态向下一个阶段的状态进行更新。

![最小路径和阶段](../img/post/dp/最小路径和阶段.png){: .center-block :}

因为在每个阶段中，我们需要知道它的具体坐标位置来进行状态转移，所以令 dp[i][j] 表示位于网格第 i 行第 j 列的最小路径和。

状态转移方程，由题目中 `每次只能向下或者向右移动一步` 这句话得到，也就是 (i, j) 可以从 (i - 1, j) 或者 (i, j - 1) 走过来，那么转移方程就是 dp[i][j] = min(dp[i - 1][j], dp[i][j - 1] + grid[i][j])。

- **记忆化搜索的代码**

```java
class Solution {
    private int[][] dp;

    public int minPathSum(int[][] grid) {
        int rows = grid.length, cols = grid[0].length;
        dp = new int[rows][cols];
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                dp[i][j] = -1;
            }
        }
        return dfs(grid, rows - 1, cols - 1);
    }

    private int dfs(int[][] grid, int i, int j) {
        // 搜不下去就是边界
        if (i == 0 && j == 0) return grid[i][j];
        if (i < 0 || j < 0) return Integer.MAX_VALUE;

        // 记忆化
        if (dp[i][j] != -1) return dp[i][j];

        // 状态转移
        dp[i][j] = Math.min(dfs(grid, i - 1, j), dfs(grid, i, j - 1)) + grid[i][j];
        return dp[i][j];
    }
}
```


- **递推代码**

如果使用递推，我们还需要考虑枚举状态的顺序。因为更新一个状态需要它左边与上边状态的答案，所以按照从上往下，从左往右的顺序枚举状态，可以保证一个状态被更新时，它需要的状态都已经更新完毕。

在递推中做不到像搜索那样，搜不下去就是边界，我们需要人为地将第 0 行与第 0 列的元素都作为决策边界。

```java
class Solution {
    public int minPathSum(int[][] grid) {
        int rows = grid.length, cols = grid[0].length;
        int[][] dp = new int[rows][cols];

        // 决策边界
        dp[0][0] = grid[0][0];
        for (int i = 1; i < rows; i++) dp[i][0] = dp[i - 1][0] + grid[i][0];
        for (int j = 1; j < cols; j++) dp[0][j] = dp[0][j - 1] + grid[0][j];

        // 状态转移
        for (int i = 1; i < rows; i++) {
            for (int j = 1; j < cols; j++) {
                dp[i][j] = Math.min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j];
            }
        }

        return dp[rows - 1][cols - 1];
    }
}
```

## 四、题目分类

### 1. 序列型

这类题目通常给定一个类似序列的东西，比如[数列](https://leetcode-cn.com/problems/climbing-stairs/)，[字符串](https://leetcode-cn.com/problems/maximum-subarray/)或者[抽象但存在先后顺序的东西](https://leetcode-cn.com/problems/house-robber/)。

对于这种类型的题目，阶段地划分就按照序列从前往后的顺序进行，考虑第 i 个位置，那么就位于阶段 i，接下来要往阶段 i + 1 转移。在设计状态的时候，也会将序列中的位置作为状态表示的一维。例如 dp[i]，而这一维一般来说可以表示以下几种信息：

- 第 i 个位置的答案。
- 前 i 个位置的答案。
- 前 i 个位置里，第 i 个位置一定选择的答案。

若题目复杂一些，还需要根据具体题目，在状态表示中拓展一些维度表示其他信息。

状态计算的顺序，就严格按照阶段顺序，我们先计算 dp[1][xxx]，再计算 dp[2][xxx]，直到最后计算出 dp[n][xxx]。

状态转移方程，大概是这种样子：

dp[i][aaa] = opt{dp[j][bbb]} + cost<sub>i</sub>, (i, aaa, j, bbb) ∈ C

其中 aaa，bbb 是额外的状态表示，它们之间可能需要满足题目的某种要求，用要求集合 C 来表示。

opt 就是视题目而定的操作，一般来说可能是 min，max 或者 ∑ 等。

cost<sub>i</sub> 就是转移到状态 i 的代价，也可能和 aaa，bbb，j 有关。

动态规划时间复杂度：状态数量 * 决策数量 * 转移代价

#### 两个技巧

{: .box-note}

1.如果使用递推实现代码，建议让动态规划有效状态的下标从 1 开始，把下标 0 让给一步决策都没有做的初始状态。通常这样做，可以让边界处理简单不少。

{: .box-note}

2.主动转移：一个状态主动找它可以转移到哪些状态；被动转移：更新一种状态的时候寻找哪些状态可以更新它。不同题目两种方式代码复杂度不同。

### 2. 坐标型 & 双序列型

这种题目的状态表示本质上就是一个二维表格，状态表示一般为 dp[i][j] 表示位于这个表格的第 i 行第 j 列的答案。

坐标型一般给定一个二维数组，然后你可以在上面按照某种规则走动，最后要求最值或者符合要求的方案数。状态表示 dp[i][j] 就是坐标 (i, j) 的答案，允许走动的规则就是状态转移的规则。 

双序列型一般给定两个字符串 S, T，在两个字符串上进行匹配操作，dp[i][j] 表示匹配完字符串 S[1 ... i]，T[1 ... j] 的答案，状态转移一般为对应匹配操作。

### 3. 划分型

划分型动态规划的特点就是给定一个序列，你需要把它不重不漏的划分为 K 个区间。这种问题一个很直接的思路就是 dp[i] 表示考虑了序列前 i 个元素。

考虑这类问题的状态转移，我们通常先枚举一个状态 dp[j]，然后考虑将序列 [j + 1, i] 接在后面变成一段，转移到状态 dp[i] （主动转移）。

如果严格分成 K 段，那么应该增加一维 [k] 表示当前分成了几段。本题型最重要的还是将序列 [j + 1, i] 接在后面变成一段这种状态转移的思想。

### 4. 状态压缩

状态压缩动态规划的特点就是使用一个数的二进制保存状态，比如：0，1，2，3 四把椅子有没有人坐，我们可以使用 [0 / 1] [0 / 1] [0 / 1] [0 / 1] 来表示，也可以使用 [0 ... 7] 来表示。

![状态压缩](../img/post/dp/状态压缩.png){: .center-block :}

这里，普及一些位运算常用写法，假设 bits 为状态压缩的数：

- 判断第 **j** 位是否为 **1**: `(bits>>j)&1`
- 将第 **j** 位变成 **1**: `bits|(1<<j)`
- 将第 **j** 为变成 **0**：`bits&~(1<<j)`

若给定题目的数据范围的某一维度非常小，那么十有八九是留给你状态压缩用的。
